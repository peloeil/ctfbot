# times チャンネル (times)

## 概要

`/times create` コマンドで `times` カテゴリ配下にテキストチャンネルを一括作成する。

## コマンド

### `/times create names`

`names` は半角カンマ（`,`）・読点（`、`）・改行のいずれかで区切って複数指定可。応答はすべて ephemeral。

処理（この順でチェックする）:
1. guild 外で実行 → 「サーバー内で実行してください。」
2. guild のカテゴリから名前が `times` のものを検索。なければ「times カテゴリが見つかりません。」
3. 各名前を正規化（lowercase、`[^a-z0-9\-]` を `-` に、連続 `-` を 1 つに、先頭末尾 `-` 除去、100 文字（`MAX_CHANNEL_NAME_LENGTH`）で切り詰め）し、順序を保って重複除去。空文字になった名前は除外
4. 有効な名前が 0 個 → 「作成するチャンネル名を入力してください。」
5. 10 個（`MAX_CHANNELS_PER_COMMAND`）を超える場合 → 「一度に作成できるチャンネルは 10 個までです。」
6. カテゴリ内の**テキストチャンネル**に同名が既存 → スキップ（voice 等は既存判定に含めない）。なければ作成

「何も作成せず中断」を保証するのは 1〜5 の事前チェックのみ。手順 6 のチャンネル作成は 1 件ずつ順に行い、途中で Discord API が失敗した場合は作成済みのチャンネルが残る（補償削除しない）。

応答:
- 作成あり: 「✅ #name1, #name2 を作成しました。」
- スキップあり: 「⏭️ #name3 は既に存在します。」
- 両方あれば両方表示

作成が 1 件以上あれば `log_audit(command_name="times create", details=["作成: #name1, #name2"])` を送信する。
